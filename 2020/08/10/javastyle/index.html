Java Google Code Style

发表于 2020-08-10 更新于 2020-08-12

源文件规范

文件名

以.java结尾

编译方式

in UTF-8

特殊字符

1. 空格使用（0x20），ASCII horizontal space character，其他的whitespace被忽略（e.g. Tab)
2. 具有转移字符如（\b, \t, \n, \f, \r, \", \' 以及 \\），只有该表达方式，而不是用octal (e.g. \012) or Unicode (e.g. \u000a) escape.

源文件结构

源文件依次包含以下结构：
1. 许可证或者版权信息（optional）
2. 包声明
3. import声明
4. 有且仅有（exactly one）top-level class
5. 每一部分之间exactly one blank line（空行）

Package Statement

package statement不受column limit的影响

Import Statement

1、 import * 是不被允许的，一定得明确说明
2、 import顺序如下：

* 所有static import（静态导入）在一个block中
* non-static import在另一个block中
* 两个block中间用blank line分割

3、static import不能用在static nested class中，得用normal import的方式引入。

static nested class: a class created insie a class, it can not access non-static data and member.

类声明（class declaration）

1、 一个source file只能有一个top-level class
2、 一个类中相同方法必须放在一起
3、 类声明的顺序没有固定的recipe，但是必须logical

Formatting

Braces（花括号）

遇到选择时使用

braces在遇到if、else、for、do、while时候使用，即使body为空，或者只有一个statemnt

对于非空的block

遵循以下规则：

1、在 { 前不换行
2、在 { 后换行
3、在 } 前换行
4、在 } 后换行，当 } 后一个语句结束了。比如 跟在else或者冒号后面的 { 不换行。

对于空block

1、 可以同非空block一样对待，也可以直接用{}(concise)
2、但是在一个multi-block statment下，不允许用concise

Block Indentation(缩进)

每次缩进为2 spaces。

One Statement Per Line

每个statement之后都有一个line break（断行）

列数限制：100

每一行最长有100个character（Unicode character point）。

Line-Wrapping

line wrapping：将一个长行分割成多行

Where to Break

1、如果在非赋值符号断开，那么断开处在符号之前，比如（. , ::, |)

2、如果在赋值符号处断开，那么断开处在符号之后，比如（=， foreach

3、一个函数名需要和之后的开括号“（”相邻

4、逗号（，）和之前的token相邻

5、箭头（->）之后断开

Indent

在line-wrapping中，后续的line必须比前一个line+4 indent
但是如果非第一行的相邻的两个line有着相似的语法结构，则用同样的缩进

Whitespace（空格）

Vertical Whitespace

空行一般出现在

1、连续的类构造器或者成员之间，比如变量、构造函数、方法、嵌入类，、静态构造器以及实例构造器。

注意：连续的field之间可以没有空格。（但是如果field之间有逻辑关系，空格是必须的。

2、源文件格式中提到的空行要求。

3、同时可以根据提升可读性要求进行。比如在第一个members/initializer 的前面加空行，或者最后一个的后面。

Horizontal Whitespace

1、保留关键字和后面的括号之间需要空格。比如 if、for、catch和之后的“（”之间需要加空格。

2、保留关键字和之前的关闭花括号需要加空格。比如else、catch与之前的“}”需要加空格。

3、在所有的open花括号“{”之前加空格。几个例外：

* @SomeAnnotation({a, b})
* String[][] x = {{"foo"}};

4、在二元或者三元operator的左右两边，都要加空格。
以及operator-like symbols:

* & 
<T extends Foo & Bar>
* | 
catch (FooException | BarException e)
* ：
for statement之间的：
* ->
（String str) -> str.length()

除了（：：）比如 Object::toString不加空格
（.）比如 object.toString()

5、在, : ;的后面，或者”)”作为cast的后面

6、在注释符号的左右两侧 //

7、between a type annotation and [] or …

8、花括号“{ }”内侧左右两边的空格可加可不加。

Horizontal Alignment

在连续注释(//)中， 上下两行的注释对齐非必需

Grouping Parentheses：recommended

Specific Constructs

初始化

Enum Classes

逗号之间的line break不是必需的，每个变量之间的空行也不是必需的。

Variable Declarations

1、每一个变量的声明都只能声明一个变量。int a, b;不允许。
例外：在for loop中的，多个变量声明是ok的(int i, j)。

2、本地的变量（local variable）一般在使用的时候，才声明。不提前声明。并且初始化在声明的时候，或者紧跟声明语句。

Arrays

Arrays的初始化可以当作 “block-like” construct.
以下几种方式皆可接受。

Switch Statement

1、和其他block一致，switch的缩进为+2
2、如果没有break，最好给个commet：fall through

Annotations(注解@)

以下方式皆可

Comments（注释）

单行

多行

Modifiers

Class和member的修饰词按照一定的顺序

Number Literals

long类型变量用“L”为后缀表示，比如 3000000L

Naming

通用原则

Google coding不用特殊前后缀，比如一下命名方式不可取

包名

所有包名都是小写，且所有字母都相邻，不存在下划线(_)等连接符号
比如com.example.deepspace, not com.example.deepSpace or com.example.deep_space

类名

类型采用大写驼峰方式（UpperCamelCase）

类和接口名一般都是采用名词或者名词短语（e.g. Character, ImmutabeList）。但是偶尔用形容词或者副词（Readable）

Test类采用类名+Test（HashTest）

方法名

方法名采用小写驼峰（lowerCamelCase）

一般是动词或者动词短语（sendMessage, stop）

变量名

变量名采用小驼峰（lowerCamelCase）。
包括

constant name

constant name 采用全大写的方式，每个word之间用下划线(_)分割。

constant指的是用 final static修饰的基础变量类型、list、immutable types、immutable collection of immutable type.

Type Variable Names

类型变量名称由两种命名方式

  • 单个大写字母（E, T, X, T2)

  • 在类名后面加T（RequestT, FooBarT）

命名定义

有时候一些固定名词改编成的类名，定义流程如下：

0、去掉单词中的奇怪符号

1、 将固定名词根据空格或者分隔符，划分成独立单词

2、所有字母变成小写，单词的第一个字母大写，而后拼凑。（注意，原单词的大小写完全忽略。

Programming Practices

@Override: always used

只要是允许的，一个方法就要以@Override表示。但如果父类方法是@Deprecated,则不要。

caught exceptions: not ignored

遇到exception要处理，如果是合理的exception，可以不要处理，但是得注释说明。

静态变量（static)

调用静态变量的时候， 必需直接用类名调用。

Javadoc

Formatting

基本格式

多行doc

单行doc

Block Tags

一般的javadoc中都包含4要素，@param, @return, @throws, @deprecated,如果某个元素一行写不下，下一行缩进+4

哪里使用Javadoc

一般javadoc在每一个public class之上，并且该类中的每个protected或者public 方法。
