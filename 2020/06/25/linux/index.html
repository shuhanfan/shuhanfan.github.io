命令与文件查询

which 查询二进制文件的位置

which是查找PATH这个环境变量中所规范的路径查询，类似cd这种内置命令无法查到，但是可以用type查询。在bash章节会提到。

文件名的查找-whereis/locate

通常find不经常用， 因为速度很慢，所以通常先用whereis或者locate来检查，如果找不到，才使用find。

  • whereis／locate 寻找特定的文件
    因为whereis和locate在寻在文件的时候，是在自己创建的数据库中寻找的（/var/lib/mlocate）不需要通过硬盘访问，所以速度更快。
    限制就是：数据库的创建默认是每天执行一次，所以如果新建文件后查找，并无法找到，需要更新数据库。

执行updatedb即可手动更新数据库。

学习shell script

shell script指的是利用shell功能所写的一个“程序”（program）。该程序使用纯文本文件，将一些shell的语法与命令写在里面。搭配正则表达式、管道命令与数据流重定向等功能，以达到我们所想要的处理目的。

1、shell.sh文件必须要具备可读与可执行（rx）权限。

2、 执行方法

* 绝对路径 /home/fsh/shell.sh
* 相对路径 ./shell.sh
* bash程序进行： bash shell.sh 或 sh shell.sh

2、 利用exit命令让程序中断，并传回一个值（$?）给系统。可以在执行完shell之后用echo $?得到返回的结果。

Script执行方式区别

1、直接执行
直接执行，不论是相对路径／绝对路径，或者是利用bash命令，都相当于开了一个bash的子进程进行执行，产生的结果不会传给父进程。

2、使用source执行
相当于在父进程中直接执行，所以变量会生效。(但是只有export 变量名才会出现在环境变量中，用env可以看到)
